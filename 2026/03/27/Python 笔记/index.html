浅拷贝与深拷贝

import copy
浅拷贝:创建新的外层容器，内部元素仍然使用原来的对象
深拷贝:
创建一个新的外层容器，并对其内部所有可变对象进行递归复制
深拷贝可以彻底消除数据之前的相互影响。
遇不可变对象不会复制，会直接引用

闭包

  1. 内层函数
  2. 被内层函数所引用的外层变量
  3. 并且外层函数要返回内层函数 只有返回了内层函数，闭包才能有意义

优点

  1. 可以记住状态，不用全局变量，也不用写类，就能在多次调用之前保存数据。
  2. 可以做配置过的函数，先传一部分参数，把环境固定仩，得到一个定制版的函数
  3. 可以实现简单的数据隐藏，外层变量对外不可见，只能通过内层函数访问
  4. 是装饰器等高级用法的基础

缺点

  1. 理解成本高
  2. 闭包里引用了很大的对象，长期不释放，可能会添加内存占用

装饰器

  1. 装饰器是一种可调用对象（通常是函数），它能接收一个函数作为参数，并且返回一个新函数
  2. 装饰器可以在不修改原函数代码的前提下，增强或改变原函数的功能

类装饰器

  1. 包含__call__方法的类，就是类装饰器
  2. 像调用函数一样，去调用类装饰器的实例对象，就会触发__call__方法的调用
  3. __call__方法通常接收一个函数作为参数，并且返回一个新函数

模块

  1. 在 Python 中， 一个.py文件就是一个模块
  2. 模块包含：变量，函数，类
  3. 通常把实现某一个特定功能代码，集中在一个模块中

分类

  1. 标准库模块
  2. 自定义模块
  3. 第三方模块

    可以控制 from 模块 import * 能导入哪些内容，值可以是列表、元组
  • _name_
    python 内置模块一个内置变量，它的具体运行方式，在主程序中运行时，输出的是__main__，从其它模块引入时，则是输出的是模块名

包

  1. 在 Python 中 包含__init__.py的文件夹就是一个包
  2. 通常将特定功能所有模块放入一个包中
  3. 可以进一步提升代码可维护性、可复用性

包与模块的关系

  1. 一个模块就是一个.py文件，包是用来管理模块的目录
  2. 一个包中可以有多个模块，也可以有子包

包的分类

  1. 标准库包
  2. 自定义包
  3. 第三方包

_init_

  1. _init_.py 是包的初始化文件，在包被导入时，init会被自动调用
  2. _init_.py 中可以编写一些包的初始化逻辑
  3. _init_.py 中所定义的内容，会被 from 包名 import * 形式全部引入
  4. _init.py 可以使用_all 控制包中哪些模块可以被from 包名 import * 引入

迭代器

  1. 能被 for 循环遍历的对象
  2. 可迭代对象能调用到__iter__方法
  3. 调用__iter__方法会得到 迭代器
  4. __iter__是一个魔法方法，当调用 iter 函数时， __iter__会自动调用
  5. 可迭代对象.iter() 等价于 iter(可迭代对象)

生成器

  1. 生成器函数：是有 yield 关键字
  2. 生成器对象：调用生成器函数时，其函数体不会立刻执行，而是返回一个生成器对象
  3. 只要函数中有yield 关键字，就是生成器函数
  4. 调用生成器对象的__next__方法，会让生成器函数中的代码开始执行
  5. 当生成器函数中的代码开始执行后，遇到yield会暂停执行，并且内部会记录暂停位置
  6. 遇到reture 会抛出 StopInteration 异常，并将 return 后面的表达式，作为异常信息
  7. yield 后面所写的表达式，会作为本次__next__方法的返回值
  8. 生成器对象是一种特殊的迭代器，通过 yield 实现了迭代器
  9. yield from 能把一个可迭代的对象，依次yield出去
  10. 使用生成器 send(值) 可以让生成器继续执行的同时把上次的值传给yield
  11. 生成器表达式，可以快速创建生成器对象 (表达式 for 变量 in 可迭代对象)

目录

os 模块
shutil 模块

  1. os.mkdir(path)：创建“单级”目录(目录已存在，则抛出异常)
  2. os.makedirs(path)：创建“多级”目录(路径中所有目录都存在，则抛出异常)
  3. os.rmdir(path)：删除空目录(如果目录不存在，或目录非空，都会抛出异常)
  4. os.removedirs(path): 递归删除空目录，在成功删除末尾一级目录后，会向上尝试把父目录（不为空）删除
  5. os.path.exists(path): 判断路径是否在（文件/目录）
  6. os.path.isdir(path): 判断是否是目录
  7. os.path.isfile(path): 判断是否是文件
  8. os.scandir(path): 扫描指定目录
  9. os.walk(path): 按层级，递归遍历指定目录下，所有的子目录和文件
  10. shutil.rmtree(): 删除所有文件及目录

GIL(Global Interpreter Lock) 全局解释器锁

GIL锁是 CPython 解释器中的一把互斥锁
无论CPU有多少核心，在某一时候，只允许进程中的一个线程执行Python代码
Cpython 解释器中的多线程模型，本质上是并发，而不是并行（是快速切换，而不是同时进行）
这样设计为了确保解释器级别的数据安全

GIL锁与Lock和RLock区别

对比维度 GIL Lock/RLock
核心本质 解释器级别的强制限制 代码级别的逻辑约定
谁来控制 Python 解释器，自动加锁、遇到I/O或超时自动释放 程序员：需要写代码 lock.acquire()和lock.release()
保护对象 保护解释器不崩，维护引用计数、垃圾回收等底层结构的线程安全 保护“数据不算错”，维护用户定义的变量、操作顺序等业务逻辑的完整性
作用范围 全局唯一：一个进程里只有一个GIL，管着所有线程 按需创建：可以创建100把不同的锁，分别保护100个不同的业务逻辑
如果没有 Python 解释器会崩溃 售票场景下，两个人可能买到同一张车票

协程

协程是一种可以被挂起，并且在挂起之后，还可以恢复执行的函数
协程又称微线程，是一种用户态内的上下文切换技术
协程不是操作系统提供的，是由程序员在用户态，实现的一种切换机制

协程函数（coroutine Function）

使用 async 关键字修饰的函数，就是协程函数
协程对象(coroutine Object):调用协程函数，就会得到协程对象

asyncio.run 方法作用

  1. 创建一个事件循环
  2. 将收的协程对象，包装成一个任务，交给事件循环
  3. 启动事件循环
  4. 会阻塞当前线程，直到任务执行完毕，并返回该任务 return 的最终结果

await

作用

  1. 挂起：await 会暂停当前协程的执行

  2. 等待：遇到 await 关键字，事件循环会立即安排后面的对象去执行，并等待对象执行完成，并且可以拿到执行结果
    关键点:
    在执行 await 后面的对象时，会出现两种情况:
    情况一:如果在执行该对象中的代码时，遇到了【await I/o操作】(需要等待外部资源返回结果的操作)例如:网络请求、文件读写等。那 CPU 的控制权就会交给事件循环、事件循环会去调度循环中的其他任务(如果有的话)
    情况二:如果该对象中的代码，不包含任何【await I/o操作】.例如:print打印、数学计算、逻辑计算等。此时事件循环拿不到 CPU 控制权，无法调度循环中的其他任务，不会发生任务切换``

  3. 恢复：当await 后的对象执行完毕，事件循环会恢复之前被挂起的协程，该协程会从当时挂起的位置继续执行，并拿到返回值。
    注意：await 后面只能写可等待对象，常见可等待对象：协程对象，Future对象，Task对象
